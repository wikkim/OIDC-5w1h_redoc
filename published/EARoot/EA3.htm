1.3.  Overview<br /></p><p><br /></p><p>The OpenID Connect protocol, in abstract, follows the following steps.<br /></p><p><br /></p><p>1.    The RP (Client) sends a request to the OpenID Provider.<br /></p><p>2.    The OP authenticates the End-User and obtains appropriate authorization.<br /></p><p>3.    The OP responds with an Access Token and an ID Token.<br /></p><p>4.    The RP can send a request with the Access Token to the UserInfo Endpoint, per Section 4.3.<br /></p><p>5.    The UserInfo Endpoint returns Claims about the End-User.<br /></p>
